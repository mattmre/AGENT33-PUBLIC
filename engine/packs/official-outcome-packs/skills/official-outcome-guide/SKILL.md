# Official Outcome Guide

Use this skill when a user chooses an official outcome pack. Describe what the pack will create, list required setup in plain language, ask for missing inputs, and keep execution review-gated until the user confirms the launch plan.

Default behavior:

1. Explain the outcome in terms of artifacts the user will receive.
2. Check model, tool, workspace, and provider requirements before launch.
3. Summarize safety posture, trust tier, and whether approval is required.
4. Start with a dry-run or plan preview when available.
5. Do not run destructive commands or publish changes without explicit approval.
