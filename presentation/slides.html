The Problem

Most agent platforms fail at the questions that matter once an agent leaves the demo notebook.

  • Who called what, with what data, against which model, on what authority?
  • How do I run this on infrastructure I control, without paying per token forever?
  • How do I isolate tenants when "multi-tenant" was bolted on after the fact?
  • How do I prove to an auditor that the same input would produce the same answer today?
  • How do I let teams ship agents without each team operating its own production?
03 / 16

The Approach

Engineered around four load-bearing decisions.

Governance First

Tool allowlists, autonomy budgets, review gates, and per-tenant scopes are part of the runtime — not an aspirational wishlist.

Evidence Over Assertion

Every invocation produces a trace, an outcome, lineage, and (optionally) a replayable artifact. Completion is something you can demonstrate, not just claim.

Multi-Tenant by Construction

Tenant ID is in every persisted row, every queue, every cache. Isolation is the default, not an add-on module.

Local-First, Remote Optional

Ollama by default; OpenAI, Anthropic, llama.cpp, vLLM, and 20+ providers swap in by environment variable. Run on a laptop or a fleet.

Agents Are Declarative

JSON definitions

  • Six reference agents ship with the platform: orchestrator, director, worker, qa, researcher, browser-agent.
  • Auto-discovered from agent-definitions/; new agents are JSON files, not Python classes.
  • Capability taxonomy (25 entries across P / I / V / R / X) catalogues what each agent can do.

Skills + tools

  • Skills are Markdown documents with frontmatter; L0/L1/L2 progressive disclosure keeps context lean.
  • Tools are JSON-Schema-validated and governance-gated per agent.
  • The tool-use loop iterates until the model produces a final answer or hits its iteration budget.

Adding an agent is a PR that adds JSON to agent-definitions/. No service restart of unrelated subsystems.

Workflows Are DAGs

YAML in, deterministic execution out. Built for retries, parallelism, and resume.

ActionWhat it does
invoke_agentCalls the agent runtime with a rendered prompt and tool context
run_commandExecutes a governed shell command through the code execution layer
validateAsserts on a response shape, value, or invariant
transformApplies an expression-based transformation to step output
conditionalBranches the DAG based on an expression result
parallel_groupFans out across N branches, joins on completion
waitBlocks for a duration or until an external signal
execute_codeRuns a sandboxed adapter (CLI today, more in roadmap)

Checkpoint persistence means a multi-hour workflow survives a restart without losing progress.

08 / 16

REST API

FastAPI server with per-tenant auth, rate limiting, SSE streaming, and signed webhooks.

Endpoints

  • POST /agents/{id}/invoke — invoke an agent
  • POST /workflows/{id}/run — start a workflow
  • GET /traces/{id} — fetch a trace
  • GET /health — liveness
  • GET /metrics — Prometheus scrape

Real-time

  • GET /agents/{id}/stream — SSE token stream
  • GET /workflows/{id}/events — SSE step events
  • POST /webhooks — HMAC-signed delivery
curl 
curl -X POST https://agent33.example.com/agents/researcher/invoke \
  -H "X-API-Key: $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"input": "Summarise the latest on hybrid retrieval", "tenant_id": "team-alpha"}'

Runtime Modes

Same image. Three operational shapes.

Lite

docker compose up

Single process plus Postgres + Ollama. Skips Redis cluster, NATS, and review automation. Designed to fit on a laptop.

Standard

docker compose up -d

Full stack: Postgres + pgvector, Redis, NATS, Ollama. Multi-tenant. Trace pipeline on. Hit-the-ground-running mid-team default.

Enterprise

kubectl apply -k

HPA-ready stateless app, durable stores, full review automation + autonomy budgets, Prometheus + OpenTelemetry wired.

Observability

If you can’t see it, you can’t ship it.

Trace pipeline

  • 10-category failure taxonomy: F-ENV, F-INP, F-EXE, F-TMO, F-RES, F-SEC, F-DEP, F-VAL, F-REV, F-UNK
  • Trace collector with retention policies
  • OpenTelemetry-compatible export
  • Per-trace replay capability

Outcomes & evaluation

  • Outcomes store records every invocation result
  • Evaluation gates compare against golden tasks
  • Regression detector catches drift
  • Cost tracking by tenant and by step
11 / 16

Security Posture

  • Tenant resolution at the edge. API key or JWT → tenant_id propagated through every layer.
  • Per-tenant scopes and rate limits. Auth middleware enforces what each caller may do.
  • Tool governance allowlists. Which tools a given agent may invoke is policy, not honor system.
  • Autonomy budgets. Per-invocation file scope, command scope, and network scope with explicit escalation.
  • Prompt injection detection. Input validation runs before tool calls execute.
  • Pack trust controls. Extension packs validate integrity and (optionally) signature before load.
  • Encryption at rest. Vault-backed secrets, Fernet-encrypted sensitive columns.

Extension Points (Declarative)

Six places to extend the platform without forking the runtime.

Build-time

  • Agent definitions (agent-definitions/*.json)
  • Workflow templates (workflows/*.yaml)
  • Skills (skills/*.md)

Runtime

  • Tool registry (Python interface or MCP server)
  • LLM provider catalog (20+ auto-registered)
  • Extension packs (signed, governed)

Each extension point has a contract; new artifacts get reviewed and governed like any production change.
