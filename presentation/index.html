Why AGENT-33?

Most agent frameworks treat governance, evidence, and multi-tenancy as add-ons. AGENT-33 treats them as first-class load-bearing subsystems wired into the runtime.

GOV

Governance First

Every tool call is JSON-Schema validated. Every execution is allowlist-gated. Autonomy budgets enforce file, command, and network scope at runtime.

EVD

Evidence Capture

Traces, lineage, checkpoints, and replay records on every run. A 10-category failure taxonomy lets operators slice incidents by class.

TEN

Multi-Tenant Native

Tenant id flows through auth middleware, every persisted row, every trace, every memory write. Cross-tenant data is a bug, not a misconfiguration.

DAG

Workflow DAGs

Composable workflows with retries, timeouts, parallel groups, conditional branches, and durable checkpoint persistence. Declarative YAML or JSON.

LCL

Local-First Runtime

Default stack runs against Ollama on a developer laptop. Every external integration is opt-in behind feature flags and graceful fallbacks.

EXT

Six Extension Points

Agents, workflows, skills, packs, tools, MCP servers. All declarative. File-drop to extend, no recompile required.

Runtime at a Glance

Stateless FastAPI surface, durable state in Postgres + Redis + NATS, optional Ollama for inference. Every subsystem is lifespan-wired and falls back to in-process when external services are unavailable.

Quick Start

Pick a path. All three end at the same FastAPI surface and operator console.

Docker Compose

Single host. Postgres, Redis, NATS, Ollama, engine, frontend.

terminal 
git clone https://github.com/\
mattmre/AGENT33-PUBLIC.git
cd AGENT33-PUBLIC/engine
cp .env.example .env
docker compose up -d
curl http://localhost:8000/health

Local Dev

Infra in Compose, engine on metal for fast iteration.

terminal 
cd engine
python -m venv .venv
. .venv/bin/activate
pip install -e ".[dev]"
docker compose up -d \
  postgres redis nats ollama
uvicorn agent33.main:app --reload

Kubernetes

Production deployment via kustomize overlays.

terminal 
cd deploy/k8s
kubectl apply -k overlays/production
kubectl -n agent33 \
  rollout status \
  deployment/agent33-api
kubectl -n agent33 \
  port-forward svc/agent33-api 8000:80

Designed For

Platform Engineering Teams

A control plane that gives you tenants, scopes, traces, and approval gates by default. Stop bolting governance on after the agents go live - bake it in from the runtime up.

AI / ML Operations

Evaluation gates, regression detection, and a release lifecycle state machine for model rollouts. Lineage and replay records make incident review tractable.

Regulated & Air-Gappable Workloads

Self-hosted Apache 2.0 software, local LLM via Ollama, opt-in egress. Run it where cloud APIs are off the table - on-prem, behind a firewall, or fully air-gapped.

Builders Shipping Multi-Tenant SaaS

Tenant id resolved at the middleware, propagated to memory, traces, and outcome records. Per-tenant rate limits, scopes, and audit trails out of the box.

What's Inside

A non-exhaustive snapshot of the engine subsystems wired into the runtime.

Subsystem Purpose
agents/ Agent registry + tool-use loop. Six reference agents auto-discovered from JSON.
workflows/ DAG executor with retries, parallel groups, conditional branches, and durable checkpoints.
skills/ SKILL.md / YAML loader with L0 / L1 / L2 progressive disclosure for system-prompt injection.
packs/ Distributable bundles of skills, agents, and tools with SHA-256 integrity and trust policy.
tools/ Schema-validated tool framework with governance, allowlists, and seven built-in tools.
execution/ Sandbox-contract code execution. CLI, Jupyter, and optional GPU Docker adapters.
llm/ Model router and provider catalog. Ollama and OpenAI-compatible adapters out of the box.
memory/ Short-term buffer + pgvector long-term store + BM25 index + RRF hybrid retrieval.
security/ JWT / API-key auth, AES-256-GCM encryption, Fernet vault, prompt-injection screening.
autonomy/ Budget lifecycle, preflight checks, runtime enforcement of file / command / network scope.
review/ Two-layer review automation with risk assessment, reviewer assignment, and sign-off state.
release/ Lifecycle state machine (planned → rc → released), pre-release checklist, rollback decisions.
evaluation/ Golden tasks, metrics calculator, gate enforcer, regression detector for release gates.
improvement/ Research intake lifecycle, lessons learned with action tracking, improvement metrics.
observability/ Structured logs, metrics, alerts, lineage, replay, and a trace pipeline with retention policies.
messaging/ NATS-backed adapters for Telegram, Discord, Slack, and WhatsApp with per-channel health checks.
knowledge/ RSS, GitHub, web, and folder ingestion adapters with APScheduler cron support.
mcp_server/ Bidirectional MCP. Hosted MCP server surface plus client integration with circuit breaker.
