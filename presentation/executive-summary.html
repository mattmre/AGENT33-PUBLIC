Agents that remember what they did.

AGENT-33 is a local-first multi-agent orchestration platform with governance, evidence capture, and multi-tenant isolation built into the runtime. It is the “real workflows” layer between a model endpoint and an operator who needs to know who did what, when, and under what authority.

“Most agent frameworks treat governance, evidence, and multi-tenancy as add-ons. AGENT-33 treats them as the load-bearing parts of the runtime.” — Design rationale

Four design pillars

1. Governance First

Every action has an identity, a tenant, and an audit trail. There is no implicit “agent does whatever the model said” path through the runtime.

  • JSON-Schema validation on every tool call
  • Per-tenant tool allowlist with autonomy budget enforcement
  • HMAC-signed one-time approval tokens for sensitive operations
  • Sandbox contract for code execution

2. Evidence Over Assertion

The platform proves what happened. Traces, lineage, and replay records survive long after the agent loop exits. Failed runs are categorised, not buried.

  • Workflow checkpoints persisted to PostgreSQL
  • Lineage records of parent-child run graphs
  • Trace pipeline with a 10-category failure taxonomy
  • Deterministic replay of recorded runs

3. Multi-Tenant by Construction

Tenant id is resolved by auth middleware and flows through every persisted row, every memory write, every trace event, and every outcome record.

  • Tenant resolution at the middleware, not the route
  • Every Alembic-managed model carries tenant_id
  • Per-tenant rate limits and scopes via JWT or API key
  • Repository methods filter by tenant by default

4. Local-First, Remote Optional

The default stack runs against Ollama on a developer laptop. Cloud providers, hosted vector databases, signing authorities, and message buses are opt-in.

  • Docker Compose bootstrap brings the entire stack up
  • Graceful fallbacks to in-process cache and message bus
  • 20+ LLM providers auto-registered from environment variables
  • Three runtime modes: lite, standard, enterprise

Where AGENT-33 fits

Platform engineering

A control plane for guarded automation across multiple tenants and squads. Tenants, scopes, approval gates, and audit trails by default.

AI / ML operations

Evaluation gates, regression detection, and a release lifecycle state machine for model rollouts. Lineage and replay records.

Regulated workloads

Self-hosted, opt-in egress, local LLM via Ollama. Runs where cloud agent platforms cannot.

Multi-tenant SaaS

Tenant id at the middleware, scopes per route, rate limits per subject. Cross-tenant data is a bug, not a misconfiguration.

Research and exploration

Packs, skills, memory, training, and improvement loops. Declarative extension points keep iteration tight.

Knowledge automation

RSS / GitHub / web / folder ingestion adapters, cron-driven knowledge refresh, hybrid BM25 + vector retrieval.

Deployment posture

Lite

SQLite + in-process cache + in-process bus, single process. Ideal for laptops, demos, and CI smoke runs.

Standard

PostgreSQL + Redis + NATS + Ollama. Docker Compose or a small Kubernetes deployment. The default operator profile.

Enterprise

Standard stack plus dedicated observability, persistent volumes, ingress, and HPA. Kustomize production overlay ships with the repo.
